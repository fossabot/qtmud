Module contents¶

qtmud’s main module contains Manager & Thing, and a few constants.

New in version 0.0.1.

Changed in version 0.0.1-features/parsing: expanded documentation

This is the main module of qtmud. It contains two objects, Thing and Manager, as well as several constants.

qtmud expects to be started by having the Manager instanced, and then that object having the add_services function called.

qtmud.NAME¶

str – the name of the MUD being presented by qtmud.

qtmud.VERSION¶

str – the version of the MUD

qtmud.HOST¶

str – the hostname the MUD should be served through

qtmud.MUD_PORT¶

int – the port the MUDSocket should bind to.

class qtmud.Manager(**kw)[source]¶

Bases: object

The manager of all qtmud things and services.

New in version 0.0.1.

When qtmud is started, it creates an instance of this class. This instance is then used to handle pretty much every game occurence. For a design overview, check the main README.

log¶

object – an instance of Logger, is used throughout the engine with qtmud.Manager.log.debug(), qtmud.Manager.log.info(), qtmud.Manager.log.warning(), qtmud.Manager.log.error(), and qtmud.Manager.log.critical().

things¶

list – a list of every thing that the manager has instanced this session.

qualities¶

dict – every quality which has been applied this session, with every thing it has been applied to as a value.

services¶

OrderedDict – an OrderedDict() of every service the manager is controlling (sending events to during tick()

subscriptions¶

dict – events and which services are looking for them

events¶

dict – the events that will occur during the next tick().

new_thing(*qualities)[source]¶

create a new thing and passes *qualities to add_qualities(thing, quality)

Parameters:*qualities (list) – A list of Qualities (classes) to be applied to the new thing.
Returns:
The thing that was created, with all
qualities applied.
Return type:object

Creates a new instance of qtmud.Thing and gives it an identity. Then, adds the thing to qtmud.Manager.things, before calling add_qualities(thing, qualities) to apply the desired qualities to the thing.

This function then returns the finished thing, with all of its qualities.

run()[source]¶

Keep the manager alive

New in version 0.0.1.

Until a KeyboardInterrupt is noticed, this function keeps qtmud calling the manager’s tick() function, keeping the game running.

class qtmud.Thing(identity, manager, **kw)[source]¶

Bases: object

The object to be instanced by manager.new_thing()

New in version 0.0.1.

Changed in version 0.0.1-features/parsing: added return of successfully changed attributes to Thing.update()

identity¶

str – the unique identifier for the instance, assigned by manager.new_thing()

manager¶

object – the thing’s manager, the same object that called new_thing()

qualities¶

list – the instances of qualities which have been applied to the thing.

Parameters:
  • identity (str) – a UUID created automatically by the manager.new_thing() function
  • manager (object) – the manager instance which is instancing the thing.

Thing anticipates being instanced by the new_thing(), which is why it expects the identity and manage parameters.
