Module contents¶

Qualities are applied to things to create complex game objects.

New in version 0.0.1.

Changed in version 0.0.1-feature/environments: Added the Renderable quality

Changed in version 0.0.2-feature/parser: Moved qualities into separate files and import them to this one, so they can all be imported from qtmud.qualities.

Every object the client interacts with, including the client itself, is a thing. What differentiates the client from a spaceship from a ham sandwich are its qualities.

Qualities are classes with an apply function that adds attributes to a thing. (The apply function is meant to be called through qtmud.Manager.add_qualities(), frequently through new_thing().

The different submodules of this package contain the qualities which qtmud uses. The Quality class outlined below is for reference/documentation purposes, and isn’t meant to be inherited or used in any way.

By adding several qualities to a thing, it’s possible to quickly make complex objects. The thing instanced when a client logs in currently has 7 qualities applied to it, adding a variety of commands. The client’s avatar is still just an instance of qtmud.Thing, but now with attributes for handling command parsing and movement and so on.

The game world is intended to be put together by creating new things and adding qualities to them to make them unique game objects. For examples of these qualities, check out the library.

The rest of this file is used to demonstrate what a basic Quality looks like, so you have something to model your development off.

Warning

Qualities which add commands must be added individually through qtmud.Manager.add_qualities(thing, [Quality]). This is a known bug and will be taken care of soon.

class qtmud.qualities.Quality[source]¶

Bases: object

This class is for API documentation purposes only. Not to be used.

New in version 0.0.1-feature/parser.

qtmud.Manager expects every Quality object to have at the very least an apply() function. This is called by qtmud.Manager.add_qualities() to add attributes to an instanced thing. These attributes might be a string representing the thing’s location, or a function letting the thing speak.

apply(thing)[source]¶

Properly applies this quality to the thing.

New in version 0.0.1-feature/parser.

Parameters:thing (object) – The thing that this quality is going to be applied to.
Returns:The thing that was passed as a parameter.
Return type:object

qtmud.Manager expects every Quality to have this function. When the manager calls add_qualities(thing, Quality), it looks for (or creates) an instance of Quality, and then calls that instance’s apply(thing), where thing is the object passed in add_quality(thing, Quality).

It’s important to note that each apply() function has a pretty verbose syntax, to prevent this function from overwriting any attributes it already has. (such as its identity, or attributes added by another Quality.)

Luckily, the syntax is the same whether your new attribute is a string or function or anything else.

if not hasattr(thing, womble): thing.womble = self.womble

If you want a function, in this case womble(), to be accessible through a command, set up the following statement:

if hasattr(thing, 'commands'):
    thing.commands['womble'] = types.MethodType(self.womble,
                                                thing)

This also means adding the line import types at the head of your Quality file. The more verbose attribute setting we use helps make sure things are able to have commands added with limited memory waste.
